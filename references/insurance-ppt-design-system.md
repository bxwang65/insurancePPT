# 保险PPT设计系统参考

> 整合 shadcn/ui + ai-atelie 的设计规范，用于指导 insurance-ppt 生成更高质量的演示文稿

---

## 一、配色系统 (shadcn/ui OKLCH风格)

基于 Gamma 风格配色，深蓝主色 + 金/青强调色：

```css
:root {
  /* 主背景 - 深蓝 */
  --bg-primary: #0D1B2A;
  --bg-surface: #1A2D42;
  --bg-card: #1E3A5F;

  /* 品牌色 */
  --brand: #C8963E;           /* 金线 - 用于CTA、强调 */
  --brand-fg: #8B6914;       /* 金色文字 */
  --accent-teal: #00D4AA;     /* 青色 - 数据高亮 */
  --accent-blue: #4FC3F7;     /* 浅蓝 - 图表 */

  /* 语义色 */
  --success: #34C759;
  --danger: #FF3B30;
  --warning: #FF9500;

  /* 文字层次 */
  --text-primary: rgba(255,255,255,0.92);
  --text-secondary: rgba(255,255,255,0.65);
  --text-muted: rgba(255,255,255,0.50);
}
```

### 图表配色（数据可视化）

| 用途 | 色值 | 说明 |
|------|------|------|
| 保证部分 | `#4FC3F7` (浅蓝) | 确定性收益 |
| 非保证部分 | `#C8963E` (金色) | 预期收益 |
| 总值 | `#00D4AA` (青色) | 总额线条 |
| 背景 | `#0D1B2A` (深蓝) | 幻灯片底色 |

## 二、排版规范 (ai-atelie typography.md)

### 字号层级

| 角色 | 字号范围 | 使用场景 |
|------|---------|---------|
| Display | 48-72px | 封面大标题 |
| H1 | 32-48px | 页面标题 |
| H2 | 24-32px | 章节标题 |
| H3 | 20-24px | 子标题 |
| Body | 14-16px | 正文内容 |
| Small | 12-13px | 注释、标签 |
| Caption | 11-12px | 微小标签 |

### 字间距规则（机器感检测关键）

- **ALL CAPS 标签**: 必须 `letter-spacing: 0.06em` 或更高
- **Display/H1 (≥32px)**: 用 `letter-spacing: -0.02em` 紧排
- **Body text**: 默认 `0`（不调整）
- **小文本 (11-13px)**: `0.01em` 到 `0.02em`

> ⚠️ ALL CAPS 无间距 = 最可靠的机器生成特征

### 行高

| 字号 | 行高 |
|------|------|
| Display/H1 (≥32px) | 1.0-1.2 |
| Body (14-18px) | 1.5-1.6 |
| Chrome (12-13px) | 1.45-1.5 |

## 三、反机器感规则 (ai-atelie anti-ai-slop.md)

### 七大禁区（P0 — 必检）

1. **❌ Tailwind indigo `#6366f1` 等作强调色**  
   → 改用 `--brand` 金色

2. **❌ 两段式"信任"渐变** (紫→蓝, 蓝→青)  
   → 平面底色 + 意图性排版

3. **❌ Emoji 替代图标** (`✨`, `🚀`, `🎯`, `⚡`, `🔥`)  
   → 用 SVG 单线图标，`currentColor` 描边

4. **❌ 无品牌字体声明** (hardcode `Inter`/`Roboto`)  
   → 使用 `--font-system` token

5. **❌ 圆角卡片 + 左侧彩色边框**（典型机器生成形状）  
   → 要么去圆角，要么去边框

6. **❌ 捏造数据** ("10倍更快"、"99.9%正常运行")  
   → 只用实际来自PDF的数据

7. **❌ 占位符文案** (`lorem ipsum`, `feature one / two`)  
   → 用真实数据或留空

### 软性警告（P1）

- `--brand` 金色在单屏出现超过2次
- 超过12个原始hex值散落在CSS中
- 每个卡片都有阴影（阴影是层级）
- 标签用 body weight 而非 uppercase + 间距

## 四、配色纪律 (ai-atelie color.md)

### 四层色彩结构

| 层级 | 占比 | 颜色 |
|------|------|------|
| **中性色** | 70-90% | 深蓝背景、表面 |
| **品牌色** | 5-10% | 金色（单一） |
| **语义色** | 0-5% | 成功/危险/警告 |
| **效果色** | <1% | 渐变、光晕 |

### 品牌色使用上限

- **单屏最多2次可见 `--brand`**  
  典型组合：①标题区的品牌标记 + ②主要CTA按钮  
  Hover 和 focus ring 算作一次

- 链接若与CTA同时出现，降级为 `--text-secondary` 下划线

## 五、对比度最低标准

| 组合 | 最低比率 |
|------|---------|
| 正文 (≤16px) on 背景 | **4.5:1** |
| 大字 (>18px 或 14px bold) | **3:1** |
| UI组件 on 相邻表面 | **3:1** |
| Focus ring on 未聚焦背景 | **3:1** |

## 六、PPT专用：页面结构规则

### 封面页
- Display 字号 48-72px，金色品牌标记
- 主标题用 negative letter-spacing
- 无装饰性渐变/几何图形

### 数据页（财富增长、关键数据）
- KPI数字用 `--accent-teal` 青色高亮
- 保证/非保证用 `--accent-blue` / `--brand` 区分
- 图表区域占比 ≤ 40% 屏面
- 留白 ≥ 30%

### 对比页
- 左右对比使用统一视觉权重
- 数据标签清晰，不堆砌
- 用 `--text-secondary` 处理次要信息

## 七、综合检查清单

生成每页PPT后自检：

- [ ] 金色（`--brand`）不超过2处
- [ ] 无 `lorem ipsum` / 占位符
- [ ] ALL CAPS标签有 `letter-spacing: 0.06em`
- [ ] 无两段式渐变背景
- [ ] 无 emoji 替代图标
- [ ] 数字列对齐使用 `font-variant-numeric: tabular-nums`
- [ ] 对比度符合最低标准
- [ ] 数据来自PDF，无捏造数字
- [ ] 无圆角+左彩边框的典型机器卡
